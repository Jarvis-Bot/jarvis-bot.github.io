Control anything from any service.

Jarvis is an open-source assistant, listening to your orders from any service.

Asciinema? YouTube?

How does it work?

Jarvis relies on addons.
There are 3 different types: sources, receivers and clients.

Sources

Sources are feeding Jarvis with data.

It can be a Tweet, a picture from Instagram, an email, or even a pressed button. Sky is the limit! If you can retrieve data from a service, an object, then it can become a source.

Receivers

Receivers are handling data. They are waiting to be triggered.

What about publishing every Instagram pic you post on Tumblr, saving them in local, and sending it to your mum via mail? Or receiving a failed notification from Travis-CI, playing an alarm sound on your computer/homeserver, and blinking every Philips Hue in your house/office?

Clients

Clients are the gate between sources and receivers.

They are used by receivers to emit something. If you need to tweet anything, you don't have to create a way to do it in the receiver, there is probably a client to do that. You can find different clients: Twitter, Tumblr, Instagram, but also Philips Hue, speakers... Anything with an input can become a client.

How to install Jarvis?

1. Clone

$ git clone https://github.com/Jarvis-Bot/Jarvis-Core.git Jarvis

2. Install

$ export PATH=Jarvis/bin:$PATH $ jarvis addons install $ jarvis

3. Enjoy!

Now that Jarvis is successfully installed, you can look for addons on Github, or craft your owns!

Don't forget to put this at the end of your .bashrc / .zshrc file :

export PATH=[PATH/TO/JARVIS/FOLDER]/bin:$PATH
